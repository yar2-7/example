Взаимодействие FlowVision с внешними программами

Функционал взаимодействия программного комплекса FlowVision с внешними программами позволяет:

1. Совместные (итерационные) FSI-расчёты

Для моделирования взаимодействия жидкости и конструкции с ПО конечно-элементного анализа в FlowVision предусмотрены коннекторы к следующим программам:

  • SIMULIA Abaqus — программа для нелинейного конечно-элементного анализа прочности и динамики конструкций.
  • АПМ* — программа для конечно-элементного анализа прочности и механики конструкций.
  • DARSYS* — программа для прочностных и динамических расчётов методом конечных элементов.
  • Fidesys* — программа для конечно-элементного моделирования в механике и физике.
  • Зенит* — программа для конечно-элементного анализа строительных конструкций.
  • Эйлер* — программа для прочностных расчётов и аэроупругости.

Также можно настроить FSI-расчет с другим ПО, подходящим для совместных расчетов, с помощью следующих интерфейсов:

  • коннектор Произвольное взаимодействие* — интерфейс для обмена данными со сторонним ПО через файлы.
  • API MBC (Moving Body Controller)* — предоставляет интерфейс для управления движением твёрдых тел из внешних программ.
  • API ВИП (Вычислительная инженерная платформа)* — предоставляет интерфейс для интеграции с инженерными расчётными пакетами.

2. Автоматизированная оптимизация

  • IOSO* — система многокритериальной оптимизации, которая может управлять расчётами в FlowVision, изменяя параметры модели для поиска оптимального решения.

3. Обработка результатов

Программы, которые получают данные от FlowVision через экспорт и работают с ними самостоятельно, без обратной связи с FlowVision:

  • EnSight — инструмент для визуализации и постобработки результатов расчётов.
  • LMS Virtual.Lab — программа для анализа шума, вибраций и акустики.
  • TORT — программа для расчёта переноса нейтронного и фотонного излучения.
* — российская разработка.
